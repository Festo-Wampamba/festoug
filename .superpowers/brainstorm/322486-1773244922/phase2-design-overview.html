Phase 2: Product Review & Rating System

Full Review Ecosystem — Design Overview

Core Decisions

Scope

Full Ecosystem — Reviews + ratings summary + helpful votes + customer dashboard + admin management

Moderation

Smart Auto-Publish — 3-5★ go live immediately; 1-2★ queued for admin approval

Eligibility

Verified Buyers Only — One review per product per customer, editable after submission

Content

Rating + Title + Body — Star rating (1-5), headline, and review text

Database Schema

-- New table
reviews
id          uuid PK
product_id   uuid FK → products
user_id      uuid FK → users
order_id     uuid FK → orders
rating       integer 1-5
title        text (5-100 chars)
content      text (10-2000 chars)
status       enum: APPROVED | PENDING | REJECTED
helpful_count integer default 0
admin_note   text nullable -- rejection reason
created_at   timestamp
updated_at   timestamp

-- New table (for helpful votes dedup)
review_helpful_votes
id         uuid PK
review_id   uuid FK → reviews
user_id     uuid FK → users
created_at  timestamp
UNIQUE(review_id, user_id)

-- Unique constraint on reviews
UNIQUE(product_id, user_id) -- one review per buyer per product

Pages & Components

Store (Customer-facing)
  • Rating summary on product page
  • Review list with sort/filter
  • "Write a Review" form
  • Helpful vote buttons
  • Verified purchase badge
Dashboard (Customer)
  • /dashboard/reviews — My Reviews
  • Edit own reviews
  • Delete own reviews
  • See rejection reasons
  • "Leave Review" on purchases
Admin
  • /admin/reviews — Manage all
  • Approve/Reject with notes
  • Filter by status/product/rating
  • Pending count on dashboard
  • Delete inappropriate reviews

API Routes

Public / Customer
GET /api/products/[id]/reviews
POST /api/reviews
PATCH /api/reviews/[id]
DELETE /api/reviews/[id]
POST /api/reviews/[id]/helpful
Admin
GET /api/admin/reviews
PATCH /api/admin/reviews/[id]
DELETE /api/admin/reviews/[id]

Customer submits review
→
Rating check
